markdown# Architecture: IA TeamPilot

Condensed version of `docs/10_System_Architecture.md`,
`docs/11_Backend_Architecture.md`, and `docs/12_Frontend_Architecture.md`.

## Layers
React Frontend
↓ HTTPS/REST (JSON)
Django REST API Layer (auth, serializers, views, permissions)
↓
Business Logic Layer (services: workload, risk, recommendation)
↓                           ↓
PostgreSQL                  AI Engine Layer (LangChain → Granite/watsonx)

## Hard Rule: Where Logic Lives
- **Numbers** (workload %, risk score) → computed in Python services
  (`apps/*/services.py`), from formulas in `docs/05_Business_Rules.md`.
  Never computed by the AI.
- **Explanations** (why a score is what it is) → generated by the AI
  Engine Layer, from data already computed above.
- The AI Engine Layer never queries the database directly. It only
  receives what a calling service explicitly passes to it.

## Backend Structure (Django)
apps/
├── accounts/         # User, UserSkill, auth
├── teams/             # Team, TeamMembership, Skill
├── projects/          # Project
├── tasks/              # Task, TaskDependency, TaskStatusHistory
├── analytics/          # WorkloadSnapshot, RiskScore (services here)
├── recommendations/    # Recommendation (trigger + ranking logic)
├── notifications/      # Notification (throttled)
└── chat/                # scoped chat request handling
ai_engine/               # framework-agnostic — no Django models here
├── langchain_client.py
├── prompts/*.txt        # versioned, never inline strings
└── chains.py
Full detail: `docs/11_Backend_Architecture.md`.

## Frontend Structure (React)
src/
├── api/          # one file per resource, JWT injection
├── auth/         # AuthContext, ProtectedRoute
├── pages/        # one folder per screen group
├── components/   # workload/, risk/, recommendations/, tasks/
└── hooks/        # useWorkload, useRiskScore, useNotifications
State: server state via lightweight fetch hooks, no Redux. Auth
state in `AuthContext`. UI state local (`useState`). Real-time
updates via polling (15–30s), not WebSockets — see
`docs/12_Frontend_Architecture.md` §6 for why.

## The Three AI Chains (never more, never fewer, without updating docs/13 first)
1. **Recommendation Justification** — explains a task reassignment.
2. **Risk Explanation** — explains a project risk score.
3. **Chat Assistant** — answers questions grounded in scoped data.

Each has an input contract, a prompt template file, an output
validator, and a deterministic fallback. Full spec:
`docs/13_AI_Architecture.md`.

## Recalculation Trigger Model
Explicit service calls from views (not Django signals) for the
primary path, so the trigger chain stays visible in code. Exception:
the 6-hour scheduled risk recalculation runs via a management
command + cron — no Celery/Redis for the MVP (deliberate simplicity
choice, see `docs/11_Backend_Architecture.md` §5).
