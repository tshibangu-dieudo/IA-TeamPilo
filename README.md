# IA TeamPilot

**AI-powered team coordination assistant** — built for the IBM AI Builders Challenge 2026, Wildcard track: *"Build Intelligent Systems for the Future of Work."*

> The AI proposes, the human decides. TeamPilot AI never applies a change automatically.

---

## 1. Overview

IA TeamPilot continuously analyzes a development team's workload and project data, detects overload and delay risks before they happen, and recommends concrete task redistributions — each one justified in plain language and requiring explicit Project Manager approval before anything changes.

## 2. The Problem

Project managers usually discover overload, blockers, and delays *after the fact* — during a standup, or worse, at the deadline. Existing tools (Jira, Trello, Asana) store task data but don't interpret it: the manager has to mentally cross-reference deadlines, availability, skills, and workload to catch a problem early. That cognitive load grows with team size and number of concurrent projects.

Full analysis: [`docs/02_Problem_Analysis.md`](docs/02_Problem_Analysis.md)

## 3. Features

- **Workload tracking** — real-time workload % per team member, computed from assigned task effort vs. capacity
- **Risk scoring** — weighted project risk score (overload, blocked tasks, deadline proximity, historical velocity)
- **AI recommendations** — reassignment suggestions ranked by skill match and resulting workload, each with a natural-language justification generated by IBM Granite
- **Human-in-the-loop approval** — every AI recommendation must be explicitly accepted or rejected by a Project Manager
- **AI chat assistant** — answers questions grounded in the user's own scoped data (no cross-project leakage)
- **Role-based dashboards** — Administrator, Project Manager, Team Member, Executive Manager each see what's relevant to them
- **Graceful degradation** — if the AI service is unavailable, the system falls back to deterministic template explanations instead of crashing

## 5. Architecture
React Frontend
↓ HTTPS/REST (JSON)
Django REST API Layer (auth, serializers, views, permissions)
↓
Business Logic Layer (services: workload, risk, recommendation)
↓                              ↓
PostgreSQL                  AI Engine Layer (LangChain → Granite/watsonx)

**Hard rule:** numeric values (workload %, risk score) are always computed by Python business rules — never by the AI model. The AI only generates the natural-language explanation for a number that's already been computed.

Full detail: [`docs/10_System_Architecture.md`](docs/10_System_Architecture.md), [`.ai/architecture.md`](.ai/architecture.md)

## 6. Tech Stack

| Layer | Technology |
|---|---|
| Backend | Python 3.11+, Django, Django REST Framework, PostgreSQL |
| Auth | djangorestframework-simplejwt (JWT) |
| Frontend | React (Vite), Tailwind CSS, Axios, React Router |
| AI | LangChain (prompt orchestration), IBM Granite via watsonx.ai |
| Dev tools | pytest, black, ruff |

**Deliberately not used:** Celery/Redis, WebSockets, vector search/RAG, Redux — see [`.ai/tech-stack.md`](.ai/tech-stack.md) for the reasoning behind each exclusion.

## 7. Installation

### Prerequisites
- Python 3.11+
- Node.js 18+
- PostgreSQL 15+

### Backend
```bash
cd backend
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements/local.txt
cp .env.example .env            # fill in DB credentials, JWT secret, watsonx API key
python manage.py migrate
python manage.py runserver
```

### Frontend
```bash
cd frontend
npm install
cp .env.example .env            # fill in API base URL
npm run dev
```

## 10. Usage

1. Log in as an Administrator to set up your organization, teams, and skills.
2. As a Project Manager, create a project and its tasks, assigning effort estimates and required skills.
3. TeamPilot AI analyzes workload and risk automatically as data changes (and every 6 hours minimum).
4. Review AI recommendations on the dashboard — accept or reject each one.
5. Use the chat assistant for quick questions about your team's current state.

## 13. IBM watsonx / Granite Integration

TeamPilot AI uses **IBM Granite**, served through **watsonx.ai**, orchestrated with **LangChain**, for exactly three purposes:

1. **Recommendation justification** — explains why a reassignment is suggested
2. **Risk explanation** — explains why a project's risk score is what it is
3. **Chat assistant** — answers scoped questions about the user's own data

The AI never queries the database directly and never modifies data. Full spec: [`docs/13_AI_Architecture.md`](docs/13_AI_Architecture.md)
